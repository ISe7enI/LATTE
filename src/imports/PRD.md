## 1. 产品概述

### 1.1 产品定位
拿铁是一款专注于力量训练记录的专业健身APP，主打"硬核记录、数据驱动"的理念。与市面上大多数健身APP不同，我们不追求社交属性，而是专注于帮助用户精确记录每一次训练，通过数据分析驱动训练进步。

### 1.2 目标用户
| 用户类型 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 健身新手 | 刚开始接触力量训练，需要指导和学习正确动作 | 动作教学、基础记录 |
| 进阶训练者 | 有一定训练基础，追求数据记录和进步追踪 | 详细记录、数据分析 |
| 资深爱好者 | 训练经验丰富，需要高级功能和自定义能力 | 自定义计划、深度分析 |
| 私人教练 | 需要为学员记录训练，管理多个客户数据 | 教练模式、多用户管理 |

### 1.3 核心卖点
1. **精准记录**：精确到每一组的重量、次数
2. **数据分析**：多维度统计训练数据，可视化展示进步曲线
3. **专业计划**：内置经典训练计划，支持自定义周期化训练
4. **极简专注**：无广告、无社交，纯粹的记录体验

### 1.4 商业模式
- **基础版**: 完整记录功能，全程零商业插播与信息流干扰
- **Pro版**: 一次性买断制（暂不开放，显示"即将推出"）
  - 智能计划（多款训练计划）
  - 饮食记录
  - Apple Watch支持
  - 丰富动作库
  - 详细报表
  - 数据图表
  - 训练卡片（分享图）
  - 教练模式

## 2. 术语定义

| 术语 | 定义 |
|-----|------|
| 1RM | One Repetition Maximum，一次最大重复重量 |
| 容量 | 训练总量，计算公式：重量 × 次数 |
| 超级组 | 连续完成两个或以上动作，中间无休息。A动作1组→B动作1组→休息算作1个超级组 |
| 递减组 | 同一动作连续做多组不同重量算作1个"递减组单元" |
| 力量容量 | 杠铃/哑铃训练容量 + 自重训练容量 |
| 有氧容量 | 独立建表统计，不参与力量容量计算 |

## 3. 核心功能模块

### 3.1 训练记录模块 (Training Logger)

#### 3.1.1 功能清单

| 功能名称 | 优先级 | 功能描述 |
|---------|-------|---------|
| 空训练创建 | P0 | 用户可快速开始一次无计划的训练，选择训练肌群后进入记录界面 |
| 计划模板训练 | P0 | 基于预设模板开始训练，自动加载计划中的动作和组数 |
| 组次记录 | P0 | 精确记录每组的重量(kg/lbs)、次数 |
| 组间计时器 | P0 | 自定义休息时间，训练完成后自动提醒（声音+震动），支持暂停/继续 |
| 超级组/递减组 | P1 | 支持复合动作记录，可设置多个动作为一组超级组 |
| 训练笔记 | P1 | 记录当天状态、感受、备注，支持文字输入 |
| 最小化悬浮 | P1 | 训练可后台运行，通过灵动岛/Live Activity快速返回记录界面 |
| 提前填写模式 | P0 | 训练开始前填写本动作要做的重量和组数，完成后打勾即可 |
| 事后补记模式 | P0 | 一整组动作完成后一次性勾选完成 |

#### 3.1.2 业务流程

```
1. 用户选择训练模式（空训练/计划训练）
2. 选择/添加训练动作
3. 【提前填写模式】训练前填写目标重量和组数
   【事后补记模式】边练边记录
4. 完成每组后记录实际重量、次数（或打勾确认）
5. 点击完成，自动开始组间计时
6. 计时结束提醒（声音+震动），开始下一组
7. 所有动作完成后，保存训练记录
```

#### 3.1.3 组间计时器规格

| 属性 | 规格 |
|-----|------|
| 提醒方式 | 声音提示（简短音效）+ 震动 |
| 计时器状态 | 用户接打电话时计时器继续运行 |
| iOS 14 Pro+ | 灵动岛显示倒计时，长按展开Live Activity操作 |
| 其他iOS设备 | 锁屏Widget同步显示倒计时 |
| 后台保持 | 播放静音音频维持后台计时服务 |
| App被杀后 | 重新打开App时提示"计时已中断" |

#### 3.1.4 超级组/递减组记录规则

**超级组**:
- A动作1组 → B动作1组 → 休息 = 1个超级组
- 容量计算: A容量 + B容量（两者之和）
- 用户可事后修改结构，将多组不同重量设置为超级组

**递减组**:
- 同一动作连续做3组不同重量 = 1个"递减组单元"
- 容量计算: 各组容量相加
- 用户可事后修改结构

#### 3.1.5 训练时长定义
训练时长 = 从开启第一个训练动作到点击"完成本次训练"的总时长，包括训练时间和组间休息时间。

#### 3.1.6 自动保存策略
- **触发条件**: 检测到App进入后台时保存
- **数据丢失**: 用户正在输入一组数据时App崩溃，这组数据允许丢失

### 3.2 训练计划模块 (Training Plans)

#### 3.2.1 功能清单

| 功能名称 | 优先级 | 功能描述 |
|---------|-------|---------|
| 官方计划库 | P0 | 内置新手、三分化、五分化、推拉腿等经典训练计划 |
| 自定义计划 | P0 | 自由创建周期性训练计划，设置每周训练日、动作、组数 |
| 计划排期 | P0 | 将计划安排到具体日期，系统自动提醒当日训练内容 |
| 计划模板 | P1 | 保存常用计划为模板，快速复用到新的周期 |
| 计划分享 | P1 | 分享计划给好友或导出为图片 |
| 计划进度 | P1 | 可视化展示计划执行进度和完成情况 |

#### 3.2.2 周期化计划规格

支持精细设置:
- 波浪周期: "第1周5×5，第2周3×3，第3周减载"
- 每周训练日配置
- 每日动作、组数、次数范围设置

#### 3.2.3 计划偏离处理
- 用户可按计划训练，也可灵活调整
- 允许"计划外动作"并标记
- 用户实时修改实际完成的重量和组数
- APP服务用户而非约束用户

### 3.3 动作库模块 (Exercise Library)

#### 3.3.1 功能清单

| 功能名称 | 优先级 | 功能描述 |
|---------|-------|---------|
| 系统动作库 | P0 | 400+标准动作，按肌群/器械/难度分类 |
| 3D动画演示 | P0 | 动作轨迹、肌肉发力点3D展示（骨骼动画） |
| 动作详解 | P0 | 动作要领、常见错误、呼吸节奏、替代动作 |
| 自定义动作 | P1 | 创建个人专属动作，上传动作视频 |
| 动作历史 | P1 | 查看某动作的所有训练记录和PR |
| 1RM估算 | P1 | 基于公式计算最大力量（Epley、Brzycki等） |
| 容量趋势 | P2 | 动作训练容量变化曲线 |

#### 3.3.2 3D动画技术规格

| 属性 | 规格 |
|-----|------|
| 动画类型 | 骨骼动画 |
| 生成方式 | AI生成（Meshy/Spline） |
| 加载策略 | 用户点击动作后加载到本地，未点击动作不加载 |
| 加载体验 | 骨架动画先播放，细节后加载 |
| 存储管理 | 自动清理（LRU淘汰旧动作） |
| 渲染引擎 | iOS SceneKit（原生方案） |

#### 3.3.3 1RM估算算法

**Epley公式**: 1RM = 重量 × (1 + 次数/30)  
**Brzycki公式**: 1RM = 重量 / (1.0278 - 0.0278 × 次数)

**可信度提示**:
- 次数 < 5次: 高可信度
- 次数 5-10次: 中等可信度
- 次数 > 10次: 低可信度（提示用户）

**实测1RM标记**:
- 递减组测极限时，最后一组RPE=9-10且次数≤3次，可标记为"实测1RM"

### 3.4 数据分析模块 (Analytics)

#### 3.4.1 功能清单

| 功能名称 | 优先级 | 功能描述 |
|---------|-------|---------|
| 训练容量 | P0 | 日/周/月容量统计（重量×次数），支持按肌群筛选 |
| 训练频次 | P0 | 每周训练次数、各肌群训练分布热力图 |
| 训练时长 | P1 | 单次训练时长趋势，平均组间休息时长 |
| 力量曲线 | P0 | 深蹲/卧推/硬拉等核心动作进步曲线 |
| PR里程碑 | P1 | 个人记录突破记录，支持分享 |
| 年度报表 | P1 | 年度训练总结 |
| 身体数据 | P1 | 体重、体脂率、围度记录和趋势 |

#### 3.4.2 容量计算规则

| 训练类型 | 容量计算公式 | 归属报表 |
|---------|-------------|---------|
| 杠铃/哑铃训练 | 重量 × 次数 | 力量容量 |
| 自重训练 | 体重 × 次数 | 力量容量 |
| 超级组 | A容量 + B容量 | 力量容量 |
| 递减组 | 各组容量之和 | 力量容量 |
| 有氧训练 | 独立建表统计 | 有氧容量 |

#### 3.4.3 力量曲线显示规则
- 3次以上训练显现力量曲线
- 对比基准: 和自己的历史数据对比
- 冷启动期: 显示示例数据（可交互的真实数据）
- 示例数据标注: "此为示例数据，非您的真实记录"
- 个性化示例: 根据用户体重/性别生成（体重0.8-1.2倍作为示例重量）

### 3.5 历史与日历模块 (History)

#### 3.5.1 功能清单

| 功能名称 | 优先级 | 功能描述 |
|---------|-------|---------|
| 训练日历 | P0 | 月历视图直观显示训练日期，标记训练类型 |
| 快速补录 | P0 | 补录遗漏的训练记录 |
| 复制训练 | P1 | 将某次训练复制到今天或任意日期 |
| 训练列表 | P1 | 按时间倒序展示所有训练，支持筛选搜索 |
| 批量编辑 | P2 | 修改或删除历史记录 |
| 数据导出 | P2 | 导出为Excel/PDF格式 |

#### 3.5.2 补录规则
- 补录上周训练时，覆盖已有"空训练"占位
- 批量导入: 支持Excel导入，系统猜测+用户确认匹配
- 用户也可选择在App里重新输入

#### 3.5.3 离线冲突处理

**场景**: 手机和手表同时记录同一次训练

| 冲突类型 | 处理方案 |
|---------|---------|
| 同一动作不同数据 | 作为两次独立训练保存 |
| 用户选择 | 让用户自己选择保存哪一组或两次都保存 |
| 被丢弃数据 | 进入回收站，保留7天 |
| 回收站存储 | 占用用户手机存储 |
| 批量操作 | 允许用户批量清空回收站 |
| 日历显示 | 同一天显示两个点 |

**手动标记同一训练**:
- 用户在开始训练前自行选择是新建训练还是已有训练
- 手表显示: "新建训练或加入现有训练？"
- "已有训练"列表显示: "训练 #1"、"训练 #2"

## 4. 非功能需求

### 4.1 性能需求

| 需求项 | 说明 |
|-------|------|
| 响应时间 | 页面加载时间 < 2秒，操作响应 < 200ms |
| 并发支持 | 支持10万用户同时在线 |
| 数据量 | 单用户可存储10年训练数据 |

### 4.2 兼容性需求

| 需求项 | 说明 |
|-------|------|
| iOS | 支持iOS 14及以上版本 |
| iPad | 适配iPad |
| 手表 | 未来支持Apple Watch |

**平台策略**: MVP阶段仅iOS，上线后3个月用户增长良好则启动Android

### 4.3 安全需求

| 需求项 | 说明 |
|-------|------|
| 数据加密 | 用户数据采用AES-256加密存储 |
| 传输安全 | 全站HTTPS，TLS 1.3 |
| 隐私保护 | 符合GDPR、个人信息保护法要求 |
| 健康数据 | 心率数据符合HIPAA（美国）和《个人信息保护法》（中国） |
